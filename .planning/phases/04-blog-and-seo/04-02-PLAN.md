   - Server Component. Import `sanityFetch` from `@/sanity/lib/live`, `ALL_POSTS_QUERY` from `@/sanity/lib/queries`
   - Accept `params: Promise<{ locale: string }>`, await it, call `setRequestLocale(locale)` before any async work
   - Validate locale with `hasLocale(routing.locales, locale)`, call `notFound()` if invalid
   - Fetch posts: `const { data } = await sanityFetch({ query: ALL_POSTS_QUERY, params: { language: locale } })`
   - Destructure: `const [featuredPost, ...restPosts] = data ?? []`
   - Render `<BlogListing>` passing featuredPost and restPosts
   - Handle empty state: if no posts, show a message like "Noch keine Beitrage" (localize later if needed)

2. Create `src/components/blog/FeaturedPostCard.tsx`:
   - Server Component. Takes the featured post data as props
   - Render hero-sized: large image (using `urlFor` from `@/sanity/lib/image` for the mainImage), title as h2, excerpt, date formatted with `toLocaleDateString('de-DE')`, estimated reading time (e.g., "3 Min. Lesezeit"), and a "Weiterlesen" link
   - Use `Link` from `@/i18n/navigation` for the post link: `href={'/blog/' + post.slug.current}`
   - Tailwind: full-width card, image aspect-video, padding, hover state on the link

3. Create `src/components/blog/PostCard.tsx`:
   - Server Component. Takes a post object as props
   - Show: mainImage (via urlFor, aspect-video, rounded), title, excerpt (truncated to 2 lines with line-clamp-2), date, estimated reading time, category badge if present
   - Use `Link` from `@/i18n/navigation` for the post link
   - Tailwind: card with shadow-sm, hover:shadow-md transition, rounded-lg, overflow-hidden

4. Create `src/components/blog/BlogListing.tsx`:

   - Section padding and max-w-6xl mx-auto container

   - Define custom components:
     - `types.image`: render `<figure>` with `<Image>` (urlFor for src, width 800, height 450, rounded-lg), optional `<figcaption>` from value.caption
     - `block.h2`: render `<h2>` with `id={`h2-${props.value._key}`}` class="text-2xl font-bold mt-10 mb-4 scroll-mt-24" — the scroll-mt-24 ensures the heading isn't hidden under sticky navbar when scrolled to
     - `block.h3`: render `<h3>` with `id={`h3-${props.value._key}`}` class="text-xl font-semibold mt-8 mb-3 scroll-mt-24"
     - `block.blockquote`: border-l-4 border-primary pl-4 italic my-6
     - `marks.link`: `<a>` with href, target="_blank", rel="noopener noreferrer", underline styling
   - Wrap in `<div className="prose prose-lg max-w-none">` — if Tailwind prose plugin not available, use manual typography classes instead (check if @tailwindcss/typography is installed; if not, skip prose class and apply manual spacing/font-size classes)
   - Export as named export `PortableTextRenderer`

2. Create `src/components/blog/TableOfContents.tsx`:
   - Client Component (`'use client'`)
   - Accept `headings: TocItem[]` prop (import TocItem from `@/lib/blog`)
   - Render a `<nav>` with heading "Inhalt" and a list of anchor links
   - Each link: `<a href={`#${heading.id}`}>` with text, h3 items indented (pl-3)
   - Add active heading tracking: use IntersectionObserver to observe all heading elements by ID, highlight the currently visible heading in the TOC list
   - Smooth scroll: links use `onClick` with `document.getElementById(id)?.scrollIntoView({ behavior: 'smooth' })` and `e.preventDefault()`
   - Styling: sticky top-24, border-l-2, compact text-sm, active item gets text-primary + font-medium

3. Create `src/components/blog/AuthorCard.tsx`:
   - Server Component. Accept author object with name, bio, image
   - Render: small avatar image (urlFor, 64x64, rounded-full), name as bold text, bio as text-sm text-muted-foreground
   - Wrapped in a card with border and padding

4. Create `src/app/[locale]/blog/[slug]/page.tsx`:
   - Server Component. Accept `params: Promise<{ locale: string; slug: string }>`
   - Await params, call `setRequestLocale(locale)`, validate locale
   - Fetch: `const { data } = await sanityFetch({ query: POST_BY_SLUG_QUERY, params: { slug, language: locale } })`
   - If no data (post), call `notFound()`
   - Extract headings: `extractHeadings(post.body ?? [])` from `@/lib/blog`
   - Layout: `<article>` with grid `grid-cols-1 lg:grid-cols-[1fr_300px] gap-12 max-w-5xl mx-auto px-4 py-12`
     - Left column: post header (title as h1, date + reading time + category badge, mainImage), then `<PortableTextRenderer value={post.body ?? []} />`
     - Right column (`<aside>` with `lg:sticky lg:top-24 lg:self-start space-y-6`): `<TableOfContents headings={headings} />`, appointment CTA block (link to Cal.com `https://cal.com/nestorsegura/erstgespraech` styled as a card with button), `<AuthorCard>` if author exists
   - Add Article JSON-LD: Import `JsonLd` from `@/components/seo/JsonLd` (created by Plan 04-03, but the component is simple enough to create inline if needed — check if it exists; if not, render `<script type="application/ld+json">` directly)
     - Article schema: `{ '@context': 'https://schema.org', '@type': 'Article', headline: post.title, author: { '@type': 'Person', name: post.author?.name ?? 'Nestor Segura' }, datePublished: post.publishedAt, dateModified: post._updatedAt, image: post.mainImage ? urlFor(post.mainImage).width(1200).url() : undefined }`
     - Render the JSON-LD script tag in the page JSX
   - Add `generateStaticParams`: use `client.fetch()` from `@/sanity/lib/client` (NOT sanityFetch — sanityFetch is the live client meant for runtime, not build-time static generation). Fetch `ALL_POSTS_FOR_SITEMAP_QUERY` and map to `{ locale: post.language, slug: post.slug.current }`
