<task type="auto">
  <name>Task 2: Create deploy.sh and Nginx config</name>
  <files>deploy.sh, nginx/nestorsegura.com.conf</files>
  <action>
Create `deploy.sh` at project root. Must be executable (`chmod +x`). The script:

1. Sets `APP_DIR="/var/www/nestorsegura.com"` and `PROCESS_NAME="nestorsegura.com"`
2. Uses `set -e` for fail-fast
3. Handles `deploy.sh rollback`:
   - Reads previous commit hash from `$APP_DIR/.deploy_rollback_hash`
   - Runs `git checkout <hash>`
   - Falls through to the build+restart steps below
4. Normal deploy flow:
   - Saves current `git rev-parse HEAD` to `.deploy_rollback_hash`
   - Checks Node.js version (`node --version`) with a minimum version message (>=18)
   - `git pull`
   - `npm install --omit=dev`
   - `npm run build`
   - CRITICAL: `cp -r public .next/standalone/` and `cp -r .next/static .next/standalone/.next/`
   - `pm2 restart "$PROCESS_NAME" --update-env || pm2 start ecosystem.config.js`
   - `pm2 save`
5. Prints "Deploy complete" with timestamp

Create `nginx/nestorsegura.com.conf` (a template file committed to repo, to be copied to `/etc/nginx/sites-available/` on server):

- Server block 1: listen 80, redirect all to HTTPS 301
- Server block 2: listen 443 ssl, server_name nestorsegura.com www.nestorsegura.com
  - SSL cert paths: `/etc/letsencrypt/live/nestorsegura.com/fullchain.pem` and `privkey.pem`
  - `proxy_buffering off;` (CRITICAL for Next.js streaming)
  - location / block: proxy_pass to `http://127.0.0.1:3000` (NOT localhost — IPv6 issues)
  - Standard proxy headers: Host, X-Real-IP, X-Forwarded-For, X-Forwarded-Proto
  - proxy_http_version 1.1, Upgrade/Connection headers for WebSocket support
  - Comment at top: "Copy to /etc/nginx/sites-available/ and symlink to sites-enabled/"
  - Comment: "Run: sudo certbot --nginx -d nestorsegura.com -d www.nestorsegura.com"

Run `chmod +x deploy.sh` after creating.
  </action>
