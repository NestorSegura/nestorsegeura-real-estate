    1. Create `src/i18n/routing.ts` — single source of truth for locale config (from RESEARCH.md Pattern 1):
       ```typescript
       import { defineRouting } from 'next-intl/routing'
       export const routing = defineRouting({
         locales: ['de', 'en', 'es'],
         defaultLocale: 'de',
         localePrefix: 'as-needed',
       })
       ```
       German is default per user decision (primary market). 'as-needed' means / = German, /en = English, /es = Spanish.

    2. Create `src/i18n/request.ts` — locale resolution and message loading (from RESEARCH.md):
       - Use `getRequestConfig` from `next-intl/server`
       - Use `hasLocale` from `next-intl` for type-safe locale validation
       - Return `{ locale, messages }` — BOTH are required in next-intl v4 (breaking change)
       - Load messages via dynamic import: `(await import('./messages/${locale}.json')).default`

    3. Create `src/i18n/navigation.ts` — locale-aware navigation helpers (from RESEARCH.md Pattern 9):
       - Use `createNavigation` from `next-intl/navigation`
       - Export: Link, redirect, usePathname, useRouter, getPathname
       - IMPORTANT: App code must NEVER import from `next/link` or `next/navigation` directly. Always use these helpers.

    4. Create `src/middleware.ts` (from RESEARCH.md Pattern 2):
       - Import `createMiddleware` from `next-intl/middleware`
       - Import `routing` from `./i18n/routing`
       - Export default `createMiddleware(routing)`
       - Export config with matcher: `'/((?!api|studio|_next|_vercel|.*\\..*).*)'`
       - CRITICAL: The matcher MUST exclude `studio` and `api`. This is the highest-risk point. If /studio redirects to /de/studio, the matcher is wrong.
       - File MUST be named `middleware.ts` (NOT `proxy.ts` — that is Next.js 16 only)

    5. Create `src/types/index.ts` with next-intl v4 type augmentation (from RESEARCH.md Pattern 4):
       ```typescript
       import de from '@/i18n/messages/de.json'
       declare module 'next-intl' {
         interface AppConfig {
           Messages: typeof de
           Locale: 'de' | 'en' | 'es'
         }
       }
       ```
       Note: Uses de.json (not en.json) as the type reference since German is the source language per user decision.

    6. Update `next.config.ts` to wrap with next-intl plugin:
       ```typescript
       import createNextIntlPlugin from 'next-intl/plugin'
       const withNextIntl = createNextIntlPlugin('./src/i18n/request.ts')
       // ... wrap the existing config export with withNextIntl()
       ```
       Preserve any existing next.config.ts settings from create-next-app.
  </action>
