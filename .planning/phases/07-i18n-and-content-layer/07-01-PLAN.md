  <name>Task 1: Create i18n utilities and route segment map</name>
  <files>src/i18n/utils.ts, src/i18n/routes.ts</files>
  <action>
Create `src/i18n/utils.ts` exporting:
  - `LOCALES = ['de', 'en', 'es'] as const`
  - `type Locale = typeof LOCALES[number]`
  - `DEFAULT_LOCALE: Locale = 'de'`
  - `useTranslations(locale: Locale): (key: string) => string` — dot-notation nested lookup in messages[locale]; if not a string, fall back to messages['de']; if still not found, return the key itself (so missing keys surface visibly in dev).
  - `useTranslationArray<T>(locale: Locale, key: string): T[]` — same traversal but returns arrays (for `features.items`, `problem.items`, etc.); falls back to DE array; returns `[]` if neither exists.
  - Internal `getNestedValue(obj, path)` helper that splits on `.` and walks objects (but not arrays — arrays returned as-is for the array helper to detect).

Import the three JSON files statically: `import de from '../../messages/de.json'` etc. Do NOT use dynamic imports — these must be tree-shakeable and compile-time.

Create `src/i18n/routes.ts` exporting:
  - `ROUTE_SEGMENTS: Record<string, Record<Locale, string>>` with entries:
    - `blog: { de: 'blog', en: 'blog', es: 'blog' }`
    - `analyse: { de: 'analyse', en: 'analyze', es: 'analizar' }`
  - `localizeRoute(routeKey: string, targetLocale: Locale): string` that returns `/analyse`, `/en/analyze`, `/es/analizar` — DE gets no prefix, EN/ES get `/${locale}` prefix, then `/${segment}`. If the routeKey is not in the map, fall back to `/` + prefix + routeKey.

Both files must be strictly typed. Import `Locale` into routes.ts from utils.ts (avoid circular: routes.ts may import from utils.ts but not vice-versa).

Do NOT use `any`. Use `unknown` + narrowing in getNestedValue.
  </action>
  <verify>
`npx tsc --noEmit` has zero errors for these two files. Run a quick sanity check:
```

```
(It's fine if node can't load .ts directly — tsc is the authoritative check.)

Inspect that `useTranslations('de')('nav.home')` would return `'Startseite'` based on messages/de.json structure.
  </verify>
