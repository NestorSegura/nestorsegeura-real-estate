       - Fields: language (string, readOnly, hidden — required by i18n plugin), title (string, required), slug (slug, source: 'title', required), seo (object with title string + description text), sections (array — leave `of: []` empty for now, Plan 02 will populate with block types)

    3. Create `src/sanity/schemas/documents/post.ts`:
       - defineType with name: 'post', type: 'document'
       - Fields: language (string, readOnly, hidden), title (string, required), slug (slug, source: 'title', required), publishedAt (datetime), mainImage (image with hotspot: true), excerpt (text, rows: 3), body (array of block — portable text), author (string), seo (object with title string + description text)
       - Add orderings by publishedAt descending

    4. Create `src/sanity/schemas/documents/siteSettings.ts`:
       - defineType with name: 'siteSettings', type: 'document'
       - Fields: siteName (string, required), tagline (string), defaultCtaHref (url — the global calendar booking URL that blocks can override), navigation (array of objects with label string + href string), footer (object with socialLinks array of objects with platform string + url url)
       - SiteSettings is NOT in the i18n plugin schemaTypes — it is a global singleton, not per-locale

    5. Update `src/sanity/schemas/index.ts`:
       - Import pageType, postType, siteSettingsType
       - Export them in the schemaTypes array
