  <name>Task 1: Create all eight block schemas</name>
  <files>
    src/sanity/schemas/blocks/heroSection.ts
    src/sanity/schemas/blocks/featureStrip.ts
    src/sanity/schemas/blocks/testimonialsBlock.ts
    src/sanity/schemas/blocks/ctaBlock.ts

    src/sanity/schemas/blocks/servicesBlock.ts
    src/sanity/schemas/blocks/faqBlock.ts
    src/sanity/schemas/blocks/referencesBlock.ts
  </files>
  <action>
    Create each block as a `defineType({ type: 'object' })` — NOT inline in the page array. Each block type MUST be its own file and registered in schemaTypes for TypeGen to work.

    Every block gets these shared fields (in a 'style' group):
    - `enabled`: boolean, initialValue: true (toggle block on/off without deleting)
    - `colorScheme`: string, options list ['light', 'dark'], initialValue varies by block (hero defaults 'dark', others default 'light')
    - `spacing`: string, options list ['compact', 'normal', 'spacious'], initialValue 'normal'

    Every block gets field groups: `[{ name: 'content', title: 'Content', default: true }, { name: 'style', title: 'Style' }]`

    Content fields go in 'content' group, style fields in 'style' group.

    **heroSection** (BLCK-01):
    - headline: string, required, group: 'content'
    - subheadline: text, group: 'content'
    - ctaLabel: string, group: 'content'
    - ctaHref: url (per-block CTA override — if empty, frontend uses SiteSettings.defaultCtaHref), group: 'content'
    - backgroundImage: image with hotspot: true, group: 'content'
    - variant: string, options list [{ title: 'SVG Path Animation', value: 'svgPath' }, { title: 'Background Image', value: 'backgroundImage' }, { title: 'Text Only', value: 'textOnly' }], initialValue: 'svgPath', group: 'style'
    - colorScheme defaults to 'dark' (hero is always dark/purple per user decision)

    **featureStrip** (BLCK-02):
    - title: string, required
    - features: array of objects { icon: string, title: string (required), description: text }, validation min(3).max(6)

    **testimonialsBlock** (BLCK-03):
    - title: string, required
    - testimonials: array of objects { quote: text (required), author: string (required), role: string, avatar: image with hotspot: true }, validation min(2).max(6)

    **ctaBlock** (BLCK-04):
    - headline: string, required
    - subtext: text
    - ctaLabel: string, required
    - ctaHref: url (per-block override)
    - variant: string, options list ['primary', 'secondary'], initialValue: 'primary', group: 'style'

    **problemSolutionBlock** (BLCK-05):
    - title: string, required
    - problems: array of objects { number: number, headline: string (required), description: text }, validation min(3).max(5)

    **servicesBlock** (BLCK-06):
    - title: string, required
    - services: array of objects { number: number, name: string (required), description: text, features: array of string, ctaLabel: string, ctaHref: url }, validation min(2).max(4)

    **faqBlock** (BLCK-07):
    - title: string, required
    - faqs: array of objects { question: string (required), answer: text (required) }, validation min(4).max(10)

    **referencesBlock** (BLCK-08):
    - title: string, required
    - references: array of objects { name: string (required), description: text, image: image with hotspot: true, url: url }, validation min(3).max(8)

    IMPORTANT: Use defineArrayMember for array items, not inline object definitions. This is required for proper TypeGen inference.
  </action>
  <verify>
    Run `npx tsc --noEmit` — no TypeScript errors.
    Verify all 8 files exist: `ls src/sanity/schemas/blocks/` shows 8 .ts files.
  </verify>
  <done>Eight block schema files exist, each with enabled/colorScheme/spacing style fields, content+style field groups, and appropriate validation constraints.</done>
</task>
