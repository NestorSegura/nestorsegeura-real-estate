  <action>
    1. Create `src/hooks/useRevealOnScroll.ts` ('use client' hook):
       - Takes a ref (RefObject<HTMLElement | null>) and optional options { threshold?: number, rootMargin?: string }
       - Creates IntersectionObserver that adds 'is-visible' class to elements with 'reveal' class within the ref container
       - Observes on mount, disconnects on unmount
       - Returns nothing (side-effect only hook)
       - Default threshold: 0.1, rootMargin: '0px 0px -50px 0px'

    2. Add CSS reveal classes to `src/app/globals.css` (append, do not replace existing content):
       ```css
       /* Scroll reveal animations — CSS transitions only, no motion library */
       .reveal {
         opacity: 0;
         transform: translateY(1.5rem);
         transition: opacity 0.5s ease, transform 0.5s ease;
         transition-delay: var(--reveal-delay, 0ms);
       }
       .reveal.is-visible {
         opacity: 1;
         transform: translateY(0);
       }

       /* Respect reduced motion preference */
       @media (prefers-reduced-motion: reduce) {
         .reveal {
           opacity: 1;
           transform: none;
           transition: none;
         }
       }
       ```

    3. Create `src/blocks/GradientDivider.tsx`:
       - Server component (no 'use client')
       - Props: `from: 'light' | 'dark'`, `to: 'light' | 'dark'`
       - Renders a div with height ~80px and a CSS gradient background
       - When from='dark' to='light': gradient from the dark purple (oklch(0.25 0.08 290)) to white/off-white
       - When from='light' to='dark': gradient from off-white to dark purple
       - When from === to: render null (no divider needed)
       - Use Tailwind classes where possible, inline style for the gradient colors

    4. Create `src/blocks/PageBuilder.tsx`:
       - Import all 8 block components
       - Import GradientDivider
       - Use TypeGen types from `@/types/sanity.types` for the sections prop type. Use the PageBySlugQueryResult type (or similar from TypeGen). If TypeGen types are not available, use a manual type with `_type: string` and `_key: string` plus `[key: string]: unknown`.
       - Props: `{ sections: Array<{ _type: string; _key: string; enabled?: boolean; colorScheme?: string; [key: string]: unknown }> }`
       - Filter out blocks where `enabled === false` first
       - Map over remaining blocks with a switch on `_type`:
         - case 'heroSection': return <HeroSection key={block._key} {...block} />
         - case 'featureStrip': return <FeatureStrip key={block._key} {...block} />
         - case 'testimonialsBlock': return <TestimonialsBlock key={block._key} {...block} />
         - case 'ctaBlock': return <CtaBlock key={block._key} {...block} />
         - case 'problemSolutionBlock': return <ProblemSolutionBlock key={block._key} {...block} />
         - case 'servicesBlock': return <ServicesBlock key={block._key} {...block} />
         - case 'faqBlock': return <FaqBlock key={block._key} {...block} />
         - case 'referencesBlock': return <ReferencesBlock key={block._key} {...block} />
         - default: console.warn(`[PageBuilder] Unknown block type: ${(block as any)._type}`); return null
       - Between consecutive blocks where colorScheme transitions (e.g., dark->light or light->dark), render a <GradientDivider from={prev} to={current} />
       - The GradientDivider logic: track previous block's colorScheme, if current differs, insert divider before current block

    DELETE `src/blocks/.gitkeep` if it exists (no longer needed with real files).
