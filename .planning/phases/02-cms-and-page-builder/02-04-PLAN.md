  <action>
    1. Update `src/sanity/lib/client.ts`:
       - Change `useCdn: false` to `useCdn: true` (CDN by default for production reads)
       - Keep everything else the same
       - Add export: `export const clientWithoutCdn = client.withConfig({ useCdn: false })` for generateStaticParams

    2. Create `src/sanity/lib/live.ts`:
       - Import `defineLive` from `next-sanity/live` (CRITICAL: subpath import, NOT root import)
       - Import `client` from `@/sanity/lib/client`
       - Get `token` from `process.env.SANITY_API_READ_TOKEN`
       - If no token, throw Error('Missing SANITY_API_READ_TOKEN')
       - Export `const { sanityFetch, SanityLive } = defineLive({ client, serverToken: token, browserToken: token })`

       NOTE: If `defineLive` import fails at runtime (open question from research about sanity@4 compatibility), the fallback is to keep the existing `fetch.ts` pattern and skip SanityLive. During execution, test the import first. If it fails, create a manual sanityFetch in live.ts that wraps client.fetch with cache tags, and export a no-op SanityLive component.

    3. Create `src/sanity/lib/queries.ts`:
       - Import `defineQuery` from `next-sanity` (NOT groq — defineQuery is the modern API for TypeGen)
       - Define and export these queries:

       ```
       PAGE_BY_SLUG_QUERY: fetch page by slug + language with es fallback
         *[_type == "page" && slug.current == $slug && language == $language][0]
         ?? *[_type == "page" && slug.current == $slug && language == "es"][0]
         { _id, title, slug, language, sections[]{ ..., _type, _key } }

       ALL_PAGES_QUERY: all pages for generateStaticParams
         *[_type == "page" && defined(slug.current)]{ slug, language }

       SITE_SETTINGS_QUERY: singleton settings
         *[_type == "siteSettings"][0]{ siteName, tagline, defaultCtaHref, navigation[]{ label, href, _key }, footer{ socialLinks[]{ platform, url, _key } } }

       ALL_POSTS_QUERY: blog listing
         *[_type == "post" && language == $language] | order(publishedAt desc){ _id, title, slug, publishedAt, excerpt, mainImage, author }

       POST_BY_SLUG_QUERY: single post with es fallback
         *[_type == "post" && slug.current == $slug && language == $language][0]
         ?? *[_type == "post" && slug.current == $slug && language == "es"][0]
         { _id, title, slug, language, publishedAt, mainImage, excerpt, body, author, seo }
       ```

    4. Keep `src/sanity/lib/fetch.ts` as-is for now (it will be deprecated once live.ts is confirmed working). Add a comment at the top: `// DEPRECATED: Use sanityFetch from ./live.ts instead. Kept as fallback.`
  </action>
  <verify>

  </verify>
