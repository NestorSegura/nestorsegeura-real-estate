<task type="checkpoint:human-action" gate="blocking">
  <name>Task 2: HUMAN — Create Workers Builds project, configure domain + env vars</name>
  <what-human-does>
    This is entirely dashboard work. Claude cannot click CF UI — do NOT attempt to automate any of this via wrangler or API.

    **Step 2a: Create Worker project via Git integration**
    1. Cloudflare Dashboard → Workers & Pages → Create → Import a repository
    2. Select GitHub → authorize if needed → pick the `nestorsegura-real-estate` repo
    3. Project name MUST be exactly: `realestatestrategy-eu` (matches `name` in wrangler.jsonc — see Pitfall 2 in RESEARCH.md)
    4. Production branch: `main`
    5. Build configuration:
       - Build command: `npm run build`
       - Deploy command: leave blank (default: `npx wrangler deploy`)
       - Non-production deploy command: leave blank (default: `npx wrangler versions upload`)
       - Root directory: leave blank
    6. Click "Save and Deploy" — do NOT worry if first build fails; env vars come next.

    **Step 2b: Add build-time variables and secrets**
    Go to: Workers & Pages → realestatestrategy-eu → Settings → Build → "Build variables and secrets"
    (Note: this is DIFFERENT from runtime "Variables & Secrets" — see Pitfall 1.)

    Add these — four plain vars + one encrypted secret:
    | Name | Type | Value |
    |------|------|-------|
    | `SANITY_API_READ_TOKEN` | Secret (encrypt) | [copy from local `.env.local` line `SANITY_API_READ_TOKEN=...`] |
    | `PUBLIC_SANITY_PROJECT_ID` | Plain | `0cn4widw` |
    | `PUBLIC_SANITY_DATASET` | Plain | `production` |
    | `PUBLIC_SITE_URL` | Plain | `https://realestatestrategy.eu` |
    | `NODE_VERSION` | Plain | `22` |

    Do NOT add anything in runtime "Variables & Secrets" — the site is prerendered and has no runtime Sanity calls. Leaving runtime vars empty is correct.

    **Step 2c: Trigger a fresh build**
    - Either push a trivial commit to main, or click "Retry deployment" on the failed first build.
    - Open the build logs and confirm:
      - `npm run build` runs on Node 22 (grep the log for `node --version` or `v22`)
      - `astro build` completes without "missing env var" errors
      - Deploy step completes with a success message
    - If build fails: read the log, fix the root cause (env var typo, etc.), re-trigger. Do NOT proceed to 2d until a green build exists.

    **Step 2d: Ensure workers.dev subdomain is enabled (for preview URLs)**
    Workers & Pages → realestatestrategy-eu → Settings → Domains & Routes → confirm `<anything>.<your-subdomain>.workers.dev` is enabled (Pitfall 6). If disabled, enable it — this is what branch preview URLs use.

    **Step 2e: Add custom domain (apex)**
    Workers & Pages → realestatestrategy-eu → Settings → Domains & Routes → Add → Custom Domain → enter `realestatestrategy.eu` → Add Domain.
    - CF will create A/AAAA records automatically (if the zone is on CF) and issue an Advanced Certificate.
    - Wait for certificate status = Active (typically a few minutes).

    **Step 2f: Add proxied CNAME for www, then Redirect Rule**
    CF Dashboard → [zone: realestatestrategy.eu] → DNS → Add record:
    - Type: CNAME
    - Name: `www`
    - Target: `realestatestrategy.eu` (or `@`)
    - Proxy status: Proxied (orange cloud) — REQUIRED per Pitfall 7
    - TTL: Auto

    Then: Rules → Redirect Rules → Create Rule
    - Name: `www to apex`
    - When incoming requests match: Custom filter expression OR wildcard pattern
    - Wildcard: `https://www.*`
    - Then: Redirect URL (dynamic) → `concat("https://", regex_replace(http.host, "^www\\.", ""), http.request.uri.path_and_query)`
      (Or simpler: use the "wildcard" preset with target `https://${1}` — see RESEARCH.md "www → apex Redirect Rule" section.)
    - Status code: 301
    - Preserve query string: ON
    - Deploy the rule.

    **Step 2g: Smoke-sanity check**
    In a browser or curl:
    - `curl -I https://realestatestrategy.eu/` → expect 200 (or 301/302 to locale — just not 5xx/4xx)
    - `curl -I https://www.realestatestrategy.eu/` → expect 301 to `https://realestatestrategy.eu/`

    Full smoke test happens in 10-02 — this is just a "the lights are on" check.
